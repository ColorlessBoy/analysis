— 引言 —

有些序列不收敛到任何实数，但直觉上似乎想要收敛到 \(+\infty\) 或 \(-\infty\)。 例如，序列 \[ 1, 2, 3, 4, 5, \ldots \] 直觉上应该收敛到 \(+\infty\)，而序列 \[ -1, -2, -3, -4, -5, \ldots \] 应该收敛到 \(-\infty\)。同时，序列 \[ 1, -1, 1, -1, 1, -1, \ldots \] 似乎不收敛到任何值（尽管我们以后会看到 \(+1\) 和 \(-1\) 是它的"极限点"——见后文）。 类似地，序列 \[ 1, -2, 3, -4, 5, -6, \ldots \] 不收敛到任何实数，也不像是收敛到 \(+\infty\) 或 \(-\infty\)。 为了精确表述这些概念，我们需要引入。

6.2.1：拓展实数系

6.2.1（拓展实数系）。 \(\mathbb{R}^*\) 是在实数线 \(\mathbb{R}\) 上附加两个额外元素 \(+\infty\) 和 \(-\infty\) 而得到的。 这两个元素彼此不同，也与每个实数不同。 拓展实数 \(x\) 称为， 如果它是实数；称为， 如果它等于 \(+\infty\) 或 \(-\infty\)。
注。 这个定义与第 3.6 节中有限集和无限集的概念没有直接关系，但精神上是类似的。 至于 \(+\infty\) 和 \(-\infty\) 的具体构造并不重要； 根据习题 3.2.2，例如可以令 \(+\infty := \{\mathbb{R}\}\)，\(-\infty := \{\mathbb{R} \cup \{\infty\}\}\)。

这些新符号 \(+\infty\) 和 \(-\infty\) 目前没有太多含义，因为除了等号 \(=\) 和不等号 \(\neq\) 之外我们没有操纵它们的运算。现在我们在拓展实数系上定义一些运算。

补充说明：Lean 中的拓展实数

在 Section_6_2.lean 中，我们使用 Mathlib 的 EReal 类型来表示拓展实数系。 EReal 的定义本质上是 WithBot (WithTop ℝ)， 即先在 \(\mathbb{R}\) 上附加 \(+\infty\)（⊤），再附加 \(-\infty\)（⊥）。

定义 6.2.1 对应的 Lean 定理：

/-- Definition 6.2.1 -/ theorem EReal.def (x:EReal) : (∃ (y:Real), y = x) ∨ x = ⊤ ∨ x = ⊥

此外还定义了辅助概念 EReal.IsFinite 和 EReal.IsInfinite， 分别表示拓展实数是有限的还是无穷的。

6.2.2：拓展实数的取反

6.2.2（拓展实数的取反）。 将 \(\mathbb{R}\) 上的取反运算 \(x \mapsto -x\) 扩展到 \(\mathbb{R}^*\)，定义 \[ -(+\infty) := -\infty, \qquad -(-\infty) := +\infty. \]

这样，每个拓展实数 \(x\) 都有一个取反 \(-x\)，并且 \(-(-x)\) 总是等于 \(x\)。

补充说明：Lean 中的取反

在 Mathlib 中，EReal 的取反由以下定理描述：

  • EReal.neg_of_real：对实数 \(x\)，\(-(x : \text{EReal}) = (-x : \mathbb{R})\)
  • EReal.neg_top：\(-\top = \bot\)，即 \(-(+\infty) = -\infty\)
  • EReal.neg_bot：\(-\bot = \top\)，即 \(-(-\infty) = +\infty\)

6.2.3：拓展实数的序

6.2.3（拓展实数的序）。 设 \(x\) 和 \(y\) 是拓展实数。 我们说 \(x \leq y\)（即 \(x\) 小于或等于 \(y\)），如果以下三个陈述之一成立：
  1. \(x\) 和 \(y\) 都是实数，并且作为实数有 \(x \leq y\)；
  2. \(y = +\infty\)；
  3. \(x = -\infty\)。
我们说 \(x < y\) 如果 \(x \leq y\) 且 \(x \neq y\)。 有时我们将 \(x < y\) 写作 \(y > x\)，将 \(x \leq y\) 写作 \(y \geq x\)。
补充说明：Lean 中的序

定义 6.2.3 在 Section_6_2.lean 中对应：

/-- Definition 6.2.3 (Ordering of extended reals) -/ theorem EReal.le_iff (x y:EReal) : x ≤ y ↔ (∃ (x' y':Real), x = x' ∧ y = y' ∧ x' ≤ y') ∨ y = ⊤ ∨ x = ⊥ theorem EReal.lt_iff (x y:EReal) : x < y ↔ x ≤ y ∧ x ≠ y

Mathlib 中 EReal 已经是 CompleteLinearOrder 的实例， 所以 \(\leq\) 和 \(<\) 可以直接使用。

— 关于算术运算的讨论 —

我们也可以在拓展实数系上引入其他运算，例如加法和乘法。然而这样做有些危险， 因为这些运算几乎必然无法满足我们熟悉的代数规律。例如，要定义加法， 直觉上（基于人们对无穷大的直觉）设 \(+\infty + 5 = +\infty\) 和 \(+\infty + 3 = +\infty\) 似乎是合理的， 但这意味着 \(+\infty + 5 = +\infty + 3\)，而 \(5 \neq 3\)。 因此，一旦我们试图对无穷大进行运算，消去律之类的规则就开始崩溃了。

为了避免这些问题，我们不在拓展实数系上定义除取反和序以外的任何算术运算。

6.2.6：拓展实数集合的上确界

回忆我们在定义 5.5.10 中为实数集 \(E\) 定义了上确界（最小上界）的概念， 得到的是一个拓展实数 \(\sup(E)\)。现在我们把这个概念稍作推广。

6.2.6（拓展实数集合的上确界）。 设 \(E\) 是 \(\mathbb{R}^*\) 的子集。我们按以下规则定义 \(E\) 的 \(\sup(E)\)（或）：
  1. 如果 \(E\) 包含在 \(\mathbb{R}\) 中（即 \(+\infty\) 和 \(-\infty\) 都不是 \(E\) 的元素），则令 \(\sup(E)\) 为定义 5.5.10 中所定义的值。
  2. 如果 \(E\) 包含 \(+\infty\)，则令 \(\sup(E) := +\infty\)。
  3. 如果 \(E\) 不包含 \(+\infty\) 但包含 \(-\infty\)，则令 \(\sup(E) := \sup(E \setminus \{-\infty\})\)（此时 \(E \setminus \{-\infty\}\) 是 \(\mathbb{R}\) 的子集，属于情形 (a)）。

我们还定义 \(E\) 的 \(\inf(E)\)（也称为）： \[ \inf(E) := -\sup(-E) \] 其中 \(-E\) 是集合 \(-E := \{-x : x \in E\}\)。

补充说明：Lean 中的上确界/下确界

定义 6.2.6 的各种情形在 Section_6_2.lean 中被分别形式化：

  • EReal.sup_of_bounded_nonempty：情形 (a)，有界非空集的上确界等于实数意义下的上确界
  • EReal.sup_of_unbounded_nonempty：无界非空集的上确界为 \(+\infty\)
  • EReal.sup_of_empty：空集的上确界为 \(-\infty\)
  • EReal.sup_of_infty_mem：情形 (b)，若 \(+\infty \in E\) 则 \(\sup(E) = +\infty\)
  • EReal.sup_of_neg_infty_mem：情形 (c)，\(\sup(E) = \sup(E \setminus \{-\infty\})\)
  • EReal.inf_eq_neg_sup：下确界与上确界的关系

Mathlib 中 EReal 的 sSup 和 sInf 来自其 CompleteLattice 实例。上述定理验证了 Mathlib 的定义与教材的手工定义一致。

6.2.7–6.2.10

6.2.7。 设 \(E\) 为所有负整数加上 \(-\infty\)： \[ E = \{-1, -2, -3, -4, \ldots\} \cup \{-\infty\}. \] 则 \(\sup(E) = \sup(E \setminus \{-\infty\}) = -1\)，而 \[ \inf(E) = -\sup(-E) = -(+\infty) = -\infty. \]
6.2.8。 集合 \(\{0.9, 0.99, 0.999, 0.9999, \ldots\}\) 的下确界为 \(0.9\)，上确界为 \(1\)。 注意在这种情况下，上确界并不属于集合本身，但在某种意义上从右侧"触碰"着它。
6.2.9。 集合 \(\{1, 2, 3, 4, 5, \ldots\}\) 的下确界为 \(1\)，上确界为 \(+\infty\)。
6.2.10。 设 \(E\) 为空集。则 \(\sup(E) = -\infty\)，\(\inf(E) = +\infty\)（为什么？）。 这是上确界可以小于下确界的唯一情形（为什么？）。
注。 直觉上可以这样理解上确界和下确界。想象实数线，\(+\infty\) 在最右端，\(-\infty\) 在最左端。 想象一个活塞从 \(+\infty\) 开始向左移动，直到被集合 \(E\) 挡住；它停下的位置就是 \(\sup(E)\)。 类似地，想象一个活塞从 \(-\infty\) 开始向右移动，直到被 \(E\) 挡住；它停下的位置就是 \(\inf(E)\)。 当 \(E\) 是空集时，两个活塞穿过彼此，上确界落在 \(-\infty\)，下确界落在 \(+\infty\)。

6.2.11：最小上界与最大下界

下面的定理证明了""和""这两个名称的正当性：

6.2.11。 设 \(E\) 是 \(\mathbb{R}^*\) 的子集。则以下陈述成立：
  1. 对每个 \(x \in E\)，有 \(x \leq \sup(E)\) 且 \(x \geq \inf(E)\)。
  2. 假设 \(M \in \mathbb{R}^*\) 是 \(E\) 的，即对所有 \(x \in E\) 有 \(x \leq M\)。则 \(\sup(E) \leq M\)。
  3. 假设 \(M \in \mathbb{R}^*\) 是 \(E\) 的，即对所有 \(x \in E\) 有 \(x \geq M\)。则 \(\inf(E) \geq M\)。
证明.

定理 6.2.11 的各部分在 Section_6_2.lean 中对应（均标记为 sorry，留作练习）：

  • (a) EReal.mem_ge_inf：\(x \in E \Rightarrow \inf(E) \leq x\)
  • (b) EReal.sup_le_upper：若 \(M\) 是上界，则 \(\sup(E) \leq M\)
  • (c) EReal.inf_ge_upper：若 \(M\) 是下界，则 \(\inf(E) \geq M\)

第 6.2 节虽然篇幅不长，但包含了一个重要的观念： \(+\infty\) 和 \(-\infty\) 可以被精确定义和使用，但不应赋予它们算术运算。 Tao 特意指出 \(+\infty + 5 = +\infty + 3\) 会导致消去律崩溃， 这提醒我们在处理无穷大时必须小心谨慎。

从形式化的角度来看，Mathlib 的 EReal 类型实际上确实定义了加法和乘法， 但这些运算不满足通常的代数律（例如 EReal 不是 Ring）。 这与 Tao 的保守策略（完全不定义算术运算）形成了对比—— Mathlib 选择了"定义但标注局限性"的路线，而 Tao 选择了"干脆不定义"。 两种方法各有优缺点。
