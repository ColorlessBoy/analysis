配合 Section_5_3.lean 中 Real.eq_lim 和 Real.LIM_eq_LIM 的理解

1. 为什么需要 Quotient？—— 从集合到商空间

Tao 的实数构造：实数 = 有理柯西序列的等价类。 两个序列等价（Equiv）当且仅当它们的差趋于 0 —— 这是 CauchySequence 集合上的等价关系。

核心定义

abbrev Real := Quotient CauchySequence.instSetoid

instSetoid 把两个柯西序列定义为等价 ⇔ Sequence.Equiv a b

整体 映射为一个实数 用 Quotient.mk 把整个类压成一个点 e π √2 ℝ
柯西序列 → 等价类 → 实数轴上的点

2. 三个核心定理

Quotient.ind
归纳原理：降维到代表元
apply Quotient.ind _ x
把目标从「对所有实数 P」变成「对所有柯西序列 P」。
Quotient.exact
从等式得到等价关系
Quotient.exact h
把 ⟦a⟧ = ⟦b⟧ 转换为 a ≈ b（柯西意义上的等价）。
Quotient.sound
从等价关系得到等式
apply Quotient.sound
把 a ≈ b 提升为 ⟦a⟧ = ⟦b⟧。
sound 和 exact 构成商类型与原始集合之间的 互逆

⚠️ 为什么这看起来像循环论证？

「用了 Quotient.ind 之后直接用 a 本身当 witness，感觉像在证明自己。」 这个直觉很敏锐，但这恰恰说明 Quotient 构造的本质：

  1. 实数在 Lean 里就是被定义为「柯西序列的商」（Real := Quotient CauchySequence.instSetoid）。
    「每个实数都是某个柯西序列的形式极限」在这个定义下不是一个深刻的定理，而是一个 定义一致性检查。
  2. 证明做的事是：拿到 a : CauchySequence，用 a 本身作为 witness，证明 Quotient.mk a = LIM (a:ℕ→ℚ)。
    展开 LIM 的定义，这本质上就是 ⟦a⟧ = ⟦mk' a.cauchy⟧，而 mk' a.cauchy 就是 a 的另一个写法（mk' 只是重新打包了相同的数据）。
  3. 类比理解： 设 StudentID := Quotient (λ s₁ s₂ ↦ s₁ 和 s₂ 同班)，那么「每个学号都对应一个学生」听起来也是废话 —— 因为学号本身就是从学生构造出来的。这里的 Real.eq_lim 同理。
  4. 在 Tao 的教材中实数构造需要这个定理，是因为那里实数有独立定义（Dedekind 切割或形式极限）。但在 Lean 的形式化中，「实数就是等价类」=「每个实数都是 LIM」，所以证明只是把定义展开而已。
theorem Real.eq_lim (x : Real) : ∃ (a : ℕ → ℚ), (a : Sequence).IsCauchy ∧ x = LIM a := by
    apply Quotient.ind _ x
  intro a
    use (a : ℕ → ℚ)
  observe : ((a : ℕ → ℚ) : Sequence) = a.toSequence
    rw [this, LIM_def (by convert a.cauchy)]
  refine ⟨ a.cauchy,  ?_  ⟩
    congr; ext n; simp; replace := congr($this n); simp_all
1 Quotient.ind：要证明 ∀ x : ℝ, P(x)（即存在 a 使得 x = LIM a）， 只需对任意 a' : CauchySequence 证明 P(⟦a'⟧)。 目标变成 ∀ a' : CauchySequence, ∃ a, ... ∧ (Quotient.mk a') = LIM a。
2 引入 a : CauchySequence。用 a 本身（cast 为 ℕ → ℚ）作为 witness。 目标变成：(a : ℕ → ℚ).IsCauchy ∧ (Quotient.mk a) = LIM (a : ℕ → ℚ)。
3 LIM_def ha 说：若 a 是柯西的，则 LIM a = Quotient.mk _ (mk' ha)。 这里 a.cauchy 提供了证据。重写后目标变为： (a : ℕ → ℚ).IsCauchy ∧ (Quotient.mk a) = Quotient.mk (mk' a.cauchy)。
4 剩下要证 Quotient.mk a = Quotient.mk (mk' a.cauchy)。
refine 把前半目标 (a:ℕ→ℚ).IsCauchy 用 a.cauchy 填掉， ?_ 留下后半目标。
5 由 Quotient 的泛性质，只需证明两个 CauchySequence 作为序列相等： congr 将目标变成 a = mk' a.cauchy，ext n 变成逐点相等， 用 simp 根据定义展开即可。

为什么能 congr？ 因为 Quotient.mk 是函数，代入相等参数得相等结果。 
theorem Real.LIM_eq_LIM {a b : ℕ → ℚ} (ha : (a : Sequence).IsCauchy)
    (hb : (b : Sequence).IsCauchy) : LIM a = LIM b ↔ Sequence.Equiv a b := by
  constructor
  · --- (→) 方向 ---
    intro h
        replace h := Quotient.exact h
        rwa [dif_pos ha, dif_pos hb, CauchySequence.equiv_iff] at h
  · --- (←) 方向 ---
    intro h
        apply Quotient.sound
        rwa [dif_pos ha, dif_pos hb, CauchySequence.equiv_iff]
→1 h : LIM a = LIM b。
但 ha 和 hb 已给出，所以 LIM a = ⟦mk' ha⟧，LIM b = ⟦mk' hb⟧。 应用 Quotient.exact：把 ⟦mk' ha⟧ = ⟦mk' hb⟧ 转化为 (mk' ha) ≈ (mk' hb)（即 Sequence.Equiv）。
→2 dif_pos ha 和 dif_pos hb 展开 LIM a / LIM b 中的 if 分支， 暴露出 mk' ha 和 mk' hb。
CauchySequence.equiv_iff 把 setoid 关系翻译为 Sequence.Equiv。
重写后目标就是 Sequence.Equiv a b（← 方向的假设）。
←1 h : Sequence.Equiv a b（柯西序列的等价关系）。
目标是 LIM a = LIM b。
应用 Quotient.sound 把目标变成 setoid 关系 (mk' ha) ≈ (mk' hb)。
sound 说 ⟦x⟧ = ⟦y⟧，正好匹配我们的目标。
←2 展开 LIM a / LIM b 为 mk' ha / mk' hb。 用 CauchySequence.equiv_iff 把 setoid 的 ≈ 转化为 Sequence.Equiv a b，正是 h。

5. 交互式测试卡

点击卡片翻转，测试你是否掌握了 Quotient API：

❓ 什么时候用 ind？
点击查看答案
当你需要 证明所有实数 的某个性质时。
例如 Real.eq_lim 要证每个实数 x 都 ∃ a 使得 x = LIM a。
用 ind 降到 CauchySequence 上操作。
❓ exact vs sound？
点击查看答案
exact: 从商中的等式 ⟦a⟧=⟦b⟧ 推到原始等价关系 a≈b。
sound: 从 a≈b 推到商中的等式 ⟦a⟧=⟦b⟧。
记住方向：sound 是「升」（关系到等式），exact 是「降」（等式到关系）。
❓ LIM 为什么 noncomputable？
点击查看答案
因为 LIM a 需要判断 a 是否柯西（if h: IsCauchy ...）。 判断一个序列是否柯西在 ℕ 上不是可判定的，需要 Classical.choice。 所以标记为 noncomputable。
❓ dif_pos 有什么用？
点击查看答案
展开 LIM a 定义中的 if h : IsCauchy a then ... else 0。 已知 ha 时，dif_pos ha 把 LIM a 化简为 Quotient.mk _ (mk' ha)， 暴露出内部代表元，使 exact / sound 能作用。

6. 完整证明流程图

点击每一步展开详细说明：

theorem Real.eq_lim (x : Real) : ∃ (a : ℕ → ℚ), (a : Sequence).IsCauchy ∧ x = LIM a := by
  apply Quotient.ind _ x
  intro a
  use (a : ℕ → ℚ)
  observe : ((a : ℕ → ℚ) : Sequence) = a.toSequence
  rw [this, LIM_def (by convert a.cauchy)]
  refine ⟨ a.cauchy, ?_ ⟩
  congr; ext n; simp; replace := congr($this n); simp_all
完整流程：
  1. Quotient.ind _ x — 把目标降级为「∀ a:CauchySequence, P(⟦a⟧)」
  2. intro a — 引入代表元 a
  3. use (a : ℕ → ℚ) — 用 a 本身作为存在量词的 witness
  4. 左目标 (a:ℕ→ℚ).IsCauchy 由 a.cauchy 提供
  5. 右目标 Quotient.mk a = LIM (a:ℕ→ℚ)：展开 LIM 后用 congr 变成 a = mk' a.cauchy
  6. ext n 逐点相等，simp 搞定
theorem Real.LIM_eq_LIM {a b : ℕ → ℚ} (ha : (a : Sequence).IsCauchy)
    (hb : (b : Sequence).IsCauchy) : LIM a = LIM b ↔ Sequence.Equiv a b := by
  constructor
  · intro h
    replace h := Quotient.exact h
    rwa [dif_pos ha, dif_pos hb, CauchySequence.equiv_iff] at h
  · intro h
    apply Quotient.sound
    rwa [dif_pos ha, dif_pos hb, CauchySequence.equiv_iff]
↔ 方向总结：
→ LIM a = LIM b → 展开 dif_pos → ⟦mk' ha⟧ = ⟦mk' hb⟧ → Quotient.exact → (mk' ha) ≈ (mk' hb) → CauchySequence.equiv_iff → Sequence.Equiv a b

← Sequence.Equiv a b → CauchySequence.equiv_iff → (mk' ha) ≈ (mk' hb) → Quotient.sound → ⟦mk' ha⟧ = ⟦mk' hb⟧ → 折叠 dif_pos → LIM a = LIM b

7. 附录：完整类型签名

Quotient.ind'

∀ {α} {s : Setoid α} {p : Quotient s → Prop},
    (∀ (a : α), p (Quotient.mk'' a)) → ∀ (q : Quotient s), p q

Quotient.exact'

∀ {α} {s : Setoid α} {a b : α},
    Quotient.mk'' a = Quotient.mk'' b → s a b

Quotient.sound'

∀ {α} {s : Setoid α} {a b : α},
    s a b → Quotient.mk'' a = Quotient.mk'' b
