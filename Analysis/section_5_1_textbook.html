— 第 5 章引言 —

回顾迄今为止的进展，我们已经严格构造了三个基本数系： 系 \(\mathbb{N}\)、系 \(\mathbb{Z}\) 和系 \(\mathbb{Q}\)。 我们用五条皮亚诺公理了，并假定这样的数系存在； 然后通过取的（formal differences）\(a — b\) 构造了， 再通过取的（formal quotients）\(a // b\) 构造了。

系已经足够进行大量的数学工作——例如，如果只了解， 高中数学的大部分内容也能正常运作。 然而，有一个数学的基本领域是系所不能满足的——那就是几何学 （长度、面积等的研究）。例如，两直角边均为 1 的直角三角形的斜边长为 \(\sqrt{2}\)， 这是一个（即不是），见 4.4.4。

当涉及其它几何领域（如三角学）时，情况甚至更糟：我们会遇到 \(\pi\) 或 \(\cos(1)\) 这样的数， 它们在某种意义上比 \(\sqrt{2}\) "更加"无理。（这些数被称为 （transcendental numbers），但进一步讨论远超本书的范围。） 因此，要获得一个能充分描述几何——甚至像测量直线上长度这样简单的事情—— 的数系，我们需要用系取代系。 由于和也与几何紧密相关（想想切线的斜率、曲线下的面积）， 微也需要系才能正常运作。

然而，从构造的严格方法相当困难——所需的机制比从到、 或从到的构造要更多。在之前两种构造中，任务是为数系引入一种新的代数运算 （从到引入减法，从到引入除法）。 但从到，是从""系统到""系统的跨越， 需要引入一个截然不同的概念——（limit）。 在直觉层面上相当直观，但要严格把握却极具挑战性。 （即便是 Euler 和 Newton 这样的伟大数学家也在此概念上遇到困难。 直到 19 世纪，Cauchy 和 Dedekind 等数学家才弄清楚了如何严格处理。）

在第 4.4 节中，我们探索了中的"间隙"；现在我们将用来填补这些间隙， 从而创造。系最终将与系非常相似， 但会拥有一些新的运算——特别是（supremum）， 它随后可用于以及微所需的一切。

📖 补充说明：第 5 章的核心思想

第 5 章是全书最关键的一章。它的核心思想是：

  • 用逼近： 虽然 \(\sqrt{2}\) 不是，但我们可以用任意逼近它。
  • 作为构造工具： 每个"收敛"的（称为 ）一个。
  • ： 两个 如果"收敛到同一个"，则同一个。

这种构造方法称为的（completion of a metric space）。 它不仅仅适用于从 \(\mathbb{Q}\) 构造 \(\mathbb{R}\)，也是泛函分析中的基本技巧。

5.1.1：

5.1.1（）。 设 \(m\) 是一个。一个（sequence of rational numbers） \((a_n)_{n=m}^\infty\) 是从集合 \(\{n \in \mathbb{Z} : n \geq m\}\) 到 \(\mathbb{Q}\) 的任意函数， 即将每个 \(\geq m\) 的 \(n\) 映射到一个 \(a_n\)。 更非正式地说， \((a_n)_{n=m}^\infty\) 是的集合 \(a_m, a_{m+1}, a_{m+2}, \dots\)。
例 5.1.2。 \((n^2)_{n=0}^\infty\) 是的集合 \(0, 1, 4, 9, \dots\)； \((3)_{n=0}^\infty\) 是的集合 \(3, 3, 3, \dots\)。 这些的索引从 0 开始，但当然我们也可以使从 1 或任何其他数开始。 例如， \((a_n)_{n=3}^\infty\) 表示 \(a_3, a_4, a_5, \dots\)， 因此 \((n^2)_{n=3}^\infty\) 是的集合 \(9, 16, 25, \dots\)。
📖 补充说明：的严格

5.1.1 将正式为一个函数。这避免了"等等"（\(\dots\)）这类非正式语言。 在 Lean 中，Sequence 结构体封装了起始索引 \(n_0\)、从 \(\mathbb{Z}\) 到 \(\mathbb{Q}\) 的映射， 以及一个"在 \(n_0\) 之前全为零"的 vanish 条件以避免类型依赖的复杂性。

5.1.3：\(\varepsilon\)-稳定性

我们希望将为的。为此，我们必须区分哪些是收敛的、 哪些不是。例如，

\(1.4, 1.41, 1.414, 1.4142, 1.41421, \dots\)

看起来试图收敛到某个数，\(0.1, 0.01, 0.001, 0.0001, \dots\) 也是如此。 而其他如 \(1, 2, 4, 8, 16, \dots\) 或 \(1, 0, 1, 0, 1, \dots\) 则不是。 为此，我们使用之前的 \(\varepsilon\)-接近概念。 回忆 4.3.4：两个 \(x, y\) 是 \(\varepsilon\)-接近 的， 如果 \(d(x, y) = |x - y| \leq \varepsilon\)。

5.1.3（\(\varepsilon\)-稳定性）。 设 \(\varepsilon > 0\)。一个 \((a_n)_{n=0}^\infty\) 被称为 \(\varepsilon\)-稳定的 （\(\varepsilon\)-steady），如果的每一对元素 \(a_j, a_k\) 都是 \(\varepsilon\)-接近的， 即对每个 \(j, k\) 都有 \(|a_j - a_k| \leq \varepsilon\)。
注 5.1.4。 这个在文献中并非标准；本节之后我们将不再需要它； 下面的"最终 \(\varepsilon\)-稳定性"也是如此。 我们为从 0 开始的了 \(\varepsilon\)-稳定性， 但显然我们可以对从其他索引开始的作类似的： \(a_N, a_{N+1}, \dots\) 是 \(\varepsilon\)-稳定的，如果对所有 \(j, k \geq N\) 有 \(|a_j - a_k| \leq \varepsilon\)。
例 5.1.5。 \(1, 0, 1, 0, 1, \dots\) 是 1-稳定的，但不是 1/2-稳定的。 \(0.1, 0.01, 0.001, 0.0001, \dots\) 是 0.1-稳定的，但不是 0.01-稳定的（为什么？）。 \(1, 2, 4, 8, 16, \dots\) 对任何 \(\varepsilon\) 都不是 \(\varepsilon\)-稳定的（为什么？）。 \(2, 2, 2, 2, \dots\) 对每个 \(\varepsilon > 0\) 都是 \(\varepsilon\)-稳定的。
📖 补充说明：\(\varepsilon\)-稳定性为何太强？

\(10, 0, 0, 0, 0, 0, \dots\) 是 10-稳定的，但对任何更小的 \(\varepsilon\) 都不是 \(\varepsilon\)-稳定的， 尽管该几乎立即收敛到零。因此我们需要一个更稳健的稳定性概念， 不关心的初始部分。

5.1.6：最终 \(\varepsilon\)-稳定性

5.1.6（最终 \(\varepsilon\)-稳定性）。 设 \(\varepsilon > 0\)。一个 \((a_n)_{n=0}^\infty\) 被称为 最终 \(\varepsilon\)-稳定的 （eventually \(\varepsilon\)-steady），如果存在某个 \(N \geq 0\) 使得子 \(a_N, a_{N+1}, a_{N+2}, \dots\) 是 \(\varepsilon\)-稳定的。 换言之，存在 \(N \geq 0\) 使得对所有 \(j, k \geq N\) 有 \(|a_j - a_k| \leq \varepsilon\)。
例 5.1.7。 由 \(a_n := 1/n\) 的 \(a_1, a_2, \dots\) （即 \(1, 1/2, 1/3, 1/4, \dots\)）不是 0.1-稳定的， 但是最终 0.1-稳定的：子 \(a_{10}, a_{11}, a_{12}, \dots\) （即 \(1/10, 1/11, 1/12, \dots\)）是 0.1-稳定的。 \(10, 0, 0, 0, 0, \dots\) 对任何小于 10 的 \(\varepsilon\) 都不是 \(\varepsilon\)-稳定的， 但对每个 \(\varepsilon > 0\) 都是最终 \(\varepsilon\)-稳定的（为什么？）。

5.1.8：

5.1.8（）。 一个 \((a_n)_{n=0}^\infty\) 被称为一个 ， 如果对每个 \(\varepsilon > 0\)，该 \((a_n)_{n=0}^\infty\) 都是最终 \(\varepsilon\)-稳定的。 换言之，对每个 \(\varepsilon > 0\)，存在 \(N \geq 0\) 使得对所有 \(j, k \geq N\) 有 \(d(a_j, a_k) \leq \varepsilon\)。
注 5.1.9。 目前，参数 \(\varepsilon\) 被限制为正；我们不能取 \(\varepsilon\) 为任意正的， 因为尚未构造出来。然而，一旦我们构造了，我们将看到 如果要求 \(\varepsilon\) 为而非，上述不会改变。 换言之，我们最终将证明：一个对每个 \(\varepsilon > 0\) 最终 \(\varepsilon\)-稳定， 当且仅当它对每个 \(\varepsilon > 0\) 最终 \(\varepsilon\)-稳定；见 6.1.4。 \(\varepsilon\) 和 \(\varepsilon\) 之间的这个微妙区别在长远来看并不重要， 读者不必过多关注 \(\varepsilon\) 应是哪种数。
例 5.1.10（非正式）。 考虑之前提到的 \(1.4, 1.41, 1.414, 1.4142, \dots\)。 该已经是 0.1-稳定的。如果丢弃第一个元素 1.4，剩下的 \(1.41, 1.414, 1.4142, \dots\) 是 0.01-稳定的， 这意味着原是最终 0.01-稳定的。丢弃下一个元素得到 0.001-稳定的 \(1.414, 1.4142, \dots\)；因此原是最终 0.001-稳定的。 继续这样下去，似乎该对每个 \(\varepsilon > 0\) 都是最终 \(\varepsilon\)-稳定的， 这暗示它是一个 。然而，这个讨论并不严格——例如， 我们并未精确 \(1.4, 1.41, 1.414, \dots\) 到底是什么。 一个严格的例子如下。

5.1.11：\(1/n\) 是

5.1.11。 由 \(a_n := 1/n\) 的 \(a_1, a_2, a_3, \dots\) （即 \(1, 1/2, 1/3, \dots\)）是一个 。
我们需要证明对每个 \(\varepsilon > 0\)， \(a_1, a_2, \dots\) 是最终 \(\varepsilon\)-稳定的。 设 \(\varepsilon > 0\) 是任意的。我们需要找到 \(N \geq 1\) 使得子 \(a_N, a_{N+1}, \dots\) 是 \(\varepsilon\)-稳定的。这意味着： \[ d(a_j, a_k) \leq \varepsilon \quad\text{对所有 } j, k \geq N, \] 即 \[ |1/j - 1/k| \leq \varepsilon \quad\text{对所有 } j, k \geq N. \] 由于 \(j, k \geq N\)，我们有 \(0 < 1/j, 1/k \leq 1/N\)，因此 \(|1/j - 1/k| \leq 1/N\)。因此，要使 \(|1/j - 1/k| \leq \varepsilon\)， 只需 \(1/N < \varepsilon\)，即 \(N > 1/\varepsilon\)。 而根据 4.4.1，这样的 \(N\) 总是存在的。

5.1.12（）。 设 \(M \geq 0\) 是。
  • \(a_1, a_2, \dots, a_n\) 以 \(M\) 为界， 如果对所有 \(1 \leq i \leq n\) 有 \(|a_i| \leq M\)。
  • \((a_n)_{n=1}^\infty\) 以 \(M\) 为界， 如果对所有 \(i \geq 1\) 有 \(|a_i| \leq M\)。

例 5.1.13。 \(1, -2, 3, -4\) 是（以 4 为界，或以任何 \(\geq 4\) 的 \(M\) 为界）。 但 \(1, -2, 3, -4, 5, -6, \dots\) 是。（你能证明吗？使用 4.4.1。） \(1, -1, 1, -1, \dots\) 是（例如以 1 为界），但不是 。

5.1.14：有界

5.1.14（是）。 每个 \(a_1, a_2, \dots, a_n\) 都是。
我们对 \(n\) 进行。当 \(n = 1\) 时， \(a_1\) 显然有界： 取 \(M := |a_1|\)，则对所有 \(1 \leq i \leq n\) 有 \(|a_i| \leq M\)。 假设已对某个 \(n \geq 1\) 证明了；现在证明对 \(n+1\) 也成立。 由假设，\(a_1, a_2, \dots, a_n\) 以某个 \(M \geq 0\) 为界； 特别地，它以 \(M + |a_{n+1}|\) 为界。另一方面，\(a_{n+1}\) 也以 \(M + |a_{n+1}|\) 为界。 因此 \(a_1, a_2, \dots, a_n, a_{n+1}\) 以 \(M + |a_{n+1}|\) 为界，从而有界。 完成。
注意，虽然这个论证表明每个都有界（无论有多长）， 但它并没有说明是否有界——无穷不是。

📖 补充说明：为什么选择 ？

在构造时，可以有多种等价的方法：

  • （本书采用）： 将为 的。 这种方法的优点是适用于一般的。
  • 戴德金分割（Dedekind cuts）： 将为的"分割"。 这种方法在序理论中更自然，但不太容易推广到。
  • 无穷十进制展开： 将为无穷小数。 这种方法最直观，但在处理 \(0.999\dots = 1.000\dots\) 之类的等价关系时很繁琐。

Tao 选择 方法，因为它自然地从第 4.3-4.4 节的 \(\varepsilon\)-接近概念发展而来， 并且与后续章节中的紧密衔接。

📖 补充说明：第 5.1 节的概念层次

本节引入了四个递进的概念，它们共同构成了构造的理论基础：

  1. \(\varepsilon\)-接近（两个数之间的距离）——已在第 4.3 节；
  2. \(\varepsilon\)-稳定性（中所有元素两两 \(\varepsilon\)-接近）——对整体的强约束；
  3. 最终 \(\varepsilon\)-稳定性（去掉有限项后 \(\varepsilon\)-稳定）——丢弃初始段，关注尾部；
  4. （对每个 \(\varepsilon\) 最终 \(\varepsilon\)-稳定）——"想要收敛"的。

从稳定性到最终稳定性是从全局到尾部的转变，而从最终稳定性到 是 从固定精度到任意精度的转变。Cauchy 条件的核心思想是： 的项最终变得任意接近——它们自己"凝聚"到一个点，即使这个点（） 可能不在原始空间 \(\mathbb{Q}\) 中。

📖 补充说明：Rat.Steady / Rat.steady_def / Rat.Steady.coe 为何要拆成三层

翻开 Section_5_1.lean 第 107–125 行，你会看到三个几乎一模一样的定义——同样写着 ∀ n ≥ a.n₀, ∀ m ≥ a.n₀, ε.Close (a n) (a m)， 却分成了 Rat.Steady、Rat.steady_def、Rat.Steady.coe 三个名字。 这不是冗余，而是一个层层递进的设计。

最底层的 Rat.Steady 是通用定义。它面向一般的 Sequence——一个可以从任意整数下标 n₀ 开始的序列。所以量词必须写成 ∀ n ≥ a.n₀，而不能简单地对所有 ℕ 量化。 Rat.steady_def 则只是把同一个公式包装成 lemma，方便后续 rw 引用，没有增加任何新内容。

真正的 key 是 Rat.Steady.coe。我们绝大多数例子用的都是 ℕ → ℚ 的序列，它们通过 Coe (ℕ → ℚ) Sequence 自动提升成 Sequence。但这样一来，ε.Steady a 实际展开后变成了：

∀ n ≥ (a↑:Sequence).n₀, ∀ m ≥ (a↑:Sequence).n₀, ε.Close ((a↑:Sequence) n) ((a↑:Sequence) m)

这里藏着三处琐碎但不得不处理的细节： (a:Sequence).n₀ = 0、(a:Sequence) n 在 ℤ 上从 if n ≥ 0 then … else 0 得来、以及 n 本身是 ℤ 而非 ℕ。Rat.Steady.coe 将这些统统消化掉， 给出一个清爽的等价形式 ∀ n m : ℕ, ε.Close (a n) (a m)。

它的证明是非平凡的，而且两个方向的策略完全不同。从 ℕ 版推出原始版时，输入是 n : ℤ 且 hn : n ≥ 0，必须 lift n to ℕ 降回自然数再套用条件；反过来则只需 simp 利用 Sequence.n0_coe 和 Sequence.eval_coe 即可消去上标。

搞懂了这一层，前面那些看似 trivial 的例题就有了解释。它们不是在凑数，而是在反复训练同一个动作： 见到 ε.Steady (…:Sequence)，第一步就 rw [Rat.Steady.coe] 去掉 coercion 的包袱。 常数序列、交替序列的练习把这一肌肉记忆建立起来之后，后面处理 1/n、10^k 等真正有数学内容的例子时，学生就不需要再为类型层面的琐碎分心。而第 139 行的"raw"版本则是反面教材 ——展示不借助 Rat.Steady.coe 的时候要忍受多少 n ≥ 0 和 lift 的杂音， 让学生意识到这个引理的必要性。

同样的模式在 Sequence.IsCauchy.coe（第 267 行）再次出现，风格完全一致。

📖 补充说明：例 5.1.5 中 wlog 的执行逻辑

翻开第 166 行的 wlog h : m ≤ n，证明看起来有三个分支并列，实际上只是两层嵌套： 

wlog h : m ≤ n                 ← wlog 开两个分支
· ...rwa [abs_sub_comm]         ← ══ 分支 1：侧条件（¬ m ≤ n 时的对称归约）
  rw [abs_sub_comm, abs_of_nonneg]  ← ══ 分支 2：主分支（h : m ≤ n）
  . ...positivity               ←   ├─ rw 拆出的子目标 A（主不等式）
  linarith [...]                ←   └─ rw 拆出的子目标 B（非负性条件）

从光标处查看 wlog 生成的两个目标： 

目标 1（主分支）：
n m : ℕ
h : m ≤ n
⊢ |10 ^ (-↑n - 1) - 10 ^ (-↑m - 1)| ≤ 0.1

目标 2（侧分支）：
this : ∀ (n m : ℕ), m ≤ n → |10 ^ (-↑n - 1) - 10 ^ (-↑m - 1)| ≤ 0.1
h : ¬m ≤ n    ← 即 n < m
⊢ |10 ^ (-↑n - 1) - 10 ^ (-↑m - 1)| ≤ 0.1

分支 1 的逻辑：主分支已经证明了 "当 m ≤ n 时不等式成立"， wlog 将其泛化为 this : ∀ n m, m ≤ n → |a_n - a_m| ≤ 0.1。 侧分支里 h : ¬ m ≤ n（即 n < m），所以 n ≤ m。 把参数对调传入：specialize this m n (by linarith) 得到 |a_m - a_n| ≤ 0.1，再用 abs_sub_comm（|a-b| = |b-a|）换回 |a_n - a_m| ≤ 0.1，rwa 关门。

分支 2（主分支）：在 h : m ≤ n 下， rw [abs_sub_comm, abs_of_nonneg] 把 |a_m - a_n| 的绝对值拆掉， 同时抛出一个非负性侧条件 0 ≤ a_m - a_n。 . 块处理主不等式，linarith 行处理侧条件。

所以不是三个平级分支，而是 wlog 分两支 → 主分支的 rw 再分两支。

gcongr 开始时的目标： 

10^(-(m:ℤ)-1) - 10^(-(n:ℤ)-1) ≤ 10⁻¹ - 0

gcongr 查表发现 \(A - B \leq C - D\) 可由 \(A \leq C\) 且 \(D \leq B\) 推出 （被减数更小，减数更大，则差更小），于是拆成两个子目标：

子目标 1：\(10^{-(m:ℤ)-1} \leq 10^{-1}\)

再对 \(a^x \leq a^y\) 查表：因 \(10 > 1\)，\(x \mapsto 10^x\) 单调递增，只需 \(-(m:ℤ)-1 \leq -1\)，即 \(m \geq 0\)。 grind 自动关闭。

子目标 2：\(0 \leq 10^{-(n:ℤ)-1}\)

正数的任意实数幂恒正。positivity 自动关闭。

💬 个人感悟：这些补充说明本身就是问题

花一整页篇幅来解释 \( \varepsilon \)-稳定性的定义为什么被拆成三层、为什么 wlog 生成了两个目标、 gcongr 查了哪张表——这些补充说明本身就在证明一个尴尬的事实： Lean 形式的版本在「忠实于原文」和「被类型系统接受」之间塞进了太多不属于数学本身的机械细节。

定义从教科书到 Lean 经历了两次翻译：第一次从数学语言翻译成 Sequence + Rat.Steady， 第二次从通用形式翻译成 ℕ 版本的 Rat.Steady.coe。每一次翻译都引入了一层需要理解但不增加数学内容的 概念包袱。coercion、lift、gcongr 的单调性查表……这些对理解「什么是 Cauchy 序列」 没有贡献，但对让 Lean 接受这个证明必不可少。

这大概是形式化目前最大的门槛：它在用数学之外的复杂性换取数学之内的确定性。

归根结底是没有履约。口号是「重头构建数学基础」，拿到手的东西却处处是「我已经实现好了，但它很复杂， 你要这样用才绕得过去」——coercion 链、lift 降维、gcongr 查表、wlog 的 自动泛化……每一层都在把「机器需要什么」的债转嫁给写证明的人。声称的「地基」实则是层层增补的补丁， 每一块补丁都要求用户再学一套新规则。
