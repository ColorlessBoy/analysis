在上一节中，我们了两个 的。现在，我们终于准备好构造了： 将被为 的 的。

这是我们在本书中的第三个数系构造，也是最重要的一次：

  • 从 \(\mathbb{N}\) 到 \(\mathbb{Z}\)： 形式差 \(a - b\) 的等价类——封闭减法；
  • 从 \(\mathbb{Z}\) 到 \(\mathbb{Q}\)： 形式商 \(a // b\) 的等价类——封闭除法；

📖 补充说明：CauchySequence 的内部构造（第31–64行详解）

在 Real 被定义为商类型之前，首先需要定义被商的对象——CauchySequence。 第 31–64 行围绕这个类型建立了三层转换的桥梁：

层类型特点
最上层ℕ → ℚ用户最自然的写法，如 fun n ↦ 1/(n+1)
中间层CauchySequence保证 n₀ = 0 + Cauchy 性，统一进入商构造
底层Sequence底层存储为 ℤ → ℚ，携带 n₀ 和 vanish

下面是各声明的逐条解释：

@[ext] class CauchySequence extends Sequence where zero : n₀ = 0 cauchy : toSequence.IsCauchy

CauchySequence 在 Sequence 的基础上附加两个约束： zero 要求起始下标 n₀ = 0（否则无法统一放入商类型）； cauchy 要求该序列是 Cauchy 序列。@[ext] 自动生成一个扩展性引理： 如果两个 CauchySequence 的 seq 字段相等则它们相等 （因为 zero 只能是 rfl，cauchy 是 Prop 不参与定义性相等）。

theorem CauchySequence.ext' {a b: CauchySequence} (h: a.seq = b.seq) : a = b := by apply CauchySequence.ext _ h rw [a.zero, b.zero]

手动编写的扩展性引理。CauchySequence.ext 是由 @[ext] 自动生成的， 它有两个参数（seq 相等的证明 + zero 相等的证明）。 ext' 为用户省去第二个参数：由于 a.zero 和 b.zero 都是 n₀ = 0， 将它们 rw 掉就自动相等了。

abbrev CauchySequence.mk' {a:ℕ → ℚ} (ha: (a:Sequence).IsCauchy) : CauchySequence where n₀ := 0 seq := (a:Sequence).seq vanish := by aesop zero := rfl cauchy := ha

便捷构造器：给定一个 ℕ → ℚ 的函数 a 和它 Cauchy 的证明 ha， mk' 自动填好所有字段。其中 vanish := by aesop 由自动化策略自动证明 "序列在 n₀ 之前全为零"这个技术性条件。

@[simp] theorem CauchySequence.coe_eq {a:ℕ → ℚ} (ha: (a:Sequence).IsCauchy) : (mk' ha).toSequence = (a:Sequence) := rfl

说 mk' ha 转回 Sequence 就等于原始的 (a:Sequence)。 注意它是 rfl——类型层面的定义性相等，这是因为 mk' 直接将 a 的序列数据原样存入。

instance CauchySequence.instCoeFun : CoeFun CauchySequence (fun _ ↦ ℕ → ℚ) where coe a n := a.toSequence (n:ℤ)

注册 CoeFun 实例，使得 CauchySequence 可以像函数一样使用。 当你写 a n（其中 a : CauchySequence）时，Lean 自动将其解释为 a.toSequence (n:ℤ)。注意 n 被提升为 ℤ——因为 Sequence 底层将数据存为 ℤ → ℚ 的函数。

@[simp] theorem CauchySequence.coe_to_sequence (a: CauchySequence) : ((a:ℕ → ℚ):Sequence) = a.toSequence := by apply Sequence.ext (by simp [Sequence.n0_coe, a.zero]) ext n; by_cases h:n ≥ 0 <;> simp_all rw [a.vanish]; rwa [a.zero]

一致性引理：把 CauchySequence 转成 ℕ → ℚ 再转回 Sequence，等价于直接取 a.toSequence。 证明需要处理三处杂音：n₀ 在 Sequence.n0_coe 中变为 0、 eval 在 ℤ 上的 if n ≥ 0 ... else 0、 以及 vanish 条件。

@[simp] theorem CauchySequence.coe_coe {a:ℕ → ℚ} (ha: (a:Sequence).IsCauchy) : mk' ha = a := by rfl

最后一块拼图：mk' ha 作为 ℕ → ℚ 定义上等于 a。 所以用户在写 LIM (mk' ha) 和 LIM a 时，Lean 知道它们是一样的。

搞懂这一层，就能理解为什么 LIM、Real.add_inst、Real.mul_inst 等定义中反复出现 rw [CauchySequence.coe_to_sequence] 和 convert： 这些调用都是在手动协调三层类型之间的转换。没有这些设施， 每个涉及 CauchySequence 的证明都要处理 n₀、ℤ vs ℕ、 vanish 等与技术内容无关的琐碎细节。

5.3.4：实数的加法

5.3.4（实数的加法）。 两个的和为： \[ \operatorname{LIM}_{n\to\infty} a_n + \operatorname{LIM}_{n\to\infty} b_n := \operatorname{LIM}_{n\to\infty} (a_n + b_n). \] 为了使这个有意义，我们需要验证两件事：
  1. 如果 \((a_n)\) 和 \((b_n)\) 是 ，那么 \((a_n + b_n)\) 也是 （引理 5.3.6）；
  2. 如果 \((a_n)\) 等价于 \((a'_n)\) 且 \((b_n)\) 等价于 \((b'_n)\)，那么 \(\operatorname{LIM}(a_n + b_n) = \operatorname{LIM}(a'_n + b'_n)\)，即加法在等价类上良定（引理 5.3.7）。

5.3.6：Cauchy 序列的和是 Cauchy 的

5.3.6。 如果 \((a_n)_{n=0}^\infty\) 和 \((b_n)_{n=0}^\infty\) 是 ， 那么它们的和 \((a_n + b_n)_{n=0}^\infty\) 也是 。
设 \(\varepsilon > 0\)。由于 \(a\) 和 \(b\) 是 Cauchy 的，存在 \(N_1, N_2\) 使得 \[ \forall j,k \geq N_1,\; |a_j - a_k| \leq \varepsilon/2, \qquad \forall j,k \geq N_2,\; |b_j - b_k| \leq \varepsilon/2. \] 令 \(N = \max(N_1, N_2)\)。则对任意 \(j,k \geq N\)，由三角不等式： \[ |(a_j + b_j) - (a_k + b_k)| \leq |a_j - a_k| + |b_j - b_k| \leq \varepsilon/2 + \varepsilon/2 = \varepsilon. \] 因此 \((a_n + b_n)\) 是 。

5.3.7：等价序列的和等价

5.3.7。 如果 \((a_n)\) 等价于 \((a'_n)\)，那么对任意序列 \((b_n)\)， \((a_n + b_n)\) 等价于 \((a'_n + b_n)\)。 类似地，如果 \((b_n)\) 等价于 \((b'_n)\)，那么 \((a_n + b_n)\) 等价于 \((a_n + b'_n)\)。 因此，如果 \(a\) 等价于 \(a'\) 且 \(b\) 等价于 \(b'\)， 那么 \((a_n + b_n)\) 等价于 \((a'_n + b'_n)\)。
设 \(\varepsilon > 0\)。由于 \(a\) 等价于 \(a'\)，存在 \(N\) 使得对所有 \(n \geq N\) 有 \(|a_n - a'_n| \leq \varepsilon\)。那么对同样的 \(N\) 和所有 \(n \geq N\)： \[ |(a_n + b_n) - (a'_n + b_n)| = |a_n - a'_n| \leq \varepsilon. \] 因此 \((a_n + b_n)\) 等价于 \((a'_n + b_n)\)。另一侧由加法交换律可得。 组合两部分即得最后结论。

5.3.11：代数定律

5.3.11（代数定律）。 在 \(\mathbb{R}\) 中 （由定义 5.3.1、5.3.4、5.3.9 以及以下将的加法逆元和乘法逆元），下列性质成立：
  1. 交换律： \(x + y = y + x\)，\(x \times y = y \times x\)；
  2. 结合律： \((x + y) + z = x + (y + z)\)，\((x \times y) \times z = x \times (y \times z)\)；
  3. 分配律： \(x \times (y + z) = x \times y + x \times z\)；
  4. 单位律： \(x + 0 = x\)，\(x \times 1 = x\)；
  5. 加法逆元： 存在唯一的 \((-x)\) 使得 \(x + (-x) = 0\)；
  6. 乘法逆元： 如果 \(x \neq 0\)，存在唯一的 \(x^{-1}\) 使得 \(x \times x^{-1} = 1\)；
  7. 零乘律： \(0 \times x = 0\)。
换言之，\(\mathbb{R}\) 构成一个。
证明需要逐条验证每个定律，其中大部分由序列的逐点运算和等价类的商结构直接继承。 细节见 Section_5_3.lean 中 Real.addGroup_inst、 Real.instAddCommGroup、Real.instCommMonoid、 Real.instCommRing 和 Real.instField 的构造。

5.3.12：远离零的序列

5.3.12（远离零的序列）。 一个 \((a_n)_{n=0}^\infty\) 被称为 远离零的（bounded away from zero），如果存在一个 \(c > 0\) 使得对所有 \(n\) 都有 \(|a_n| \geq c\)。
例 5.3.13。
  • 序列 \(a_n = (-1)^n\) 是远离零的（可取 \(c = 1\)）；
  • 序列 \(a_n = 10^{-n}\) 不是远离零的（因为项趋于 0）；
  • 序列 \(a_n = 1 - 10^{-n}\) 不是远离零的（因为项趋于 1，但接近 1 并不意味远离零——实际上 \(1 - 10^{-n} \to 1\) 且最终远离零，但定义 5.3.12 要求所有 \(n\) 都满足 \(|a_n| \geq c\)，而 \(a_0 = 0\) 破坏了这一点）；
  • 序列 \(a_n = 10^{n+1}\) 是远离零的（可取 \(c = 10\)）但不是。

5.3.14

5.3.14。 设 \(x\) 是一个非零的。 那么存在一个远离零的 \((a_n)\) 使得 \(x = \operatorname{LIM} a_n\)。
由于 \(x \neq 0\)，任取一个 Cauchy 序列 \((b_n)\) 使得 \(x = \operatorname{LIM} b_n\)。 由 相等的条件， \(b\) 不等价于零序列，因此存在 \(\varepsilon > 0\) 使得对任意 \(N\) 都存在 \(n \geq N\) 满足 \(|b_n| > \varepsilon\)。 由于 \(b\) 是 Cauchy 的，存在 \(M\) 使得对所有 \(j,k \geq M\) 有 \(|b_j - b_k| \leq \varepsilon/2\)。 选择某个 \(n_0 \geq M\) 使得 \(|b_{n_0}| > \varepsilon\)，则对所有 \(k \geq M\) 有 \(|b_k| \geq |b_{n_0}| - |b_{n_0} - b_k| > \varepsilon - \varepsilon/2 = \varepsilon/2\)。 定义新的序列： \[ a_n := \begin{cases} \varepsilon/2 & \text{如果 } n < n_0,\\ b_n & \text{如果 } n \geq n_0. \end{cases} \] 则 \(a\) 是 Cauchy 的、与 \(b\) 等价的、且对所有 \(n\) 有 \(|a_n| \geq \varepsilon/2\)， 因此远离零。

5.3.15

5.3.15。 如果 \((a_n)\) 是远离零的 ， 那么 \((a_n^{-1})\) 也是 。
设 \(c > 0\) 使得对所有 \(n\) 有 \(|a_n| \geq c\)。特别地，每个 \(a_n \neq 0\)， 所以 \(a_n^{-1}\) 有定义。 设 \(\varepsilon > 0\)。由于 \(a\) 是 Cauchy 的，存在 \(N\) 使得对所有 \(j,k \geq N\) 有 \(|a_j - a_k| \leq c^2 \varepsilon\)。那么对任意 \(j,k \geq N\)： \[ \begin{aligned} |a_j^{-1} - a_k^{-1}| &= \left|\frac{a_k - a_j}{a_j a_k}\right| \\ &= \frac{|a_j - a_k|}{|a_j|\cdot|a_k|} \\ &\leq \frac{|a_j - a_k|}{c^2} \\ &\leq \frac{c^2 \varepsilon}{c^2} = \varepsilon. \end{aligned} \] 因此 \((a_n^{-1})\) 是 Cauchy 的。

5.3.16：实数的倒数

5.3.16（实数的倒数）。 设 \(x\) 是一个。 如果 \(x \neq 0\)，我们选择任意一个远离零的 \((a_n)\) 使得 \(x = \operatorname{LIM} a_n\)（由引理 5.3.14 知这样的序列存在）， 然后定义 \(x^{-1} := \operatorname{LIM} (a_n^{-1})\)。 如果 \(x = 0\)，我们定义 \(0^{-1} := 0\)（约定俗成的"垃圾"值）。
注。 定义 5.3.16 需要经典逻辑（选择公理）来选择序列 \((a_n)\)， 因为"\(x \neq 0\)"是一个无法计算判定的命题。在 Lean 中， 这通过 Classical.choice 和 if h: x ≠ 0 then ... else 0 实现。

5.3.17：倒数的良定性

5.3.17。 如果 \(\operatorname{LIM} a_n = \operatorname{LIM} b_n\)， 且 \((a_n)\) 和 \((b_n)\) 都是远离零的 ， 那么 \(\operatorname{LIM} (a_n^{-1}) = \operatorname{LIM} (b_n^{-1})\)。 换言之，倒数的与序列的选取无关。
令 \(x = \operatorname{LIM} a_n = \operatorname{LIM} b_n\)。 考虑表达式： \[ P := (\operatorname{LIM} a_n^{-1}) \times (\operatorname{LIM} a_n) \times (\operatorname{LIM} b_n^{-1}). \] 一方面， \[ P = \operatorname{LIM} (a_n^{-1} a_n b_n^{-1}) = \operatorname{LIM} (b_n^{-1}) = \operatorname{LIM} b_n^{-1}. \] 另一方面， \[ P = \operatorname{LIM} (a_n^{-1}) \times (\operatorname{LIM} b_n) \times (\operatorname{LIM} b_n^{-1}) = \operatorname{LIM} (a_n^{-1}) \times \operatorname{LIM} (b_n b_n^{-1}) = \operatorname{LIM} (a_n^{-1}) \times 1 = \operatorname{LIM} a_n^{-1}. \] 因此两边相等。

— 第 5.3 节总结 —

第 5.3 节是全书的核心。我们完成了从 到 的严格构造。这个构造的关键步骤是：

  1. 将为 的 的等价类（定义 5.3.1）；
  2. 加法和乘法，并验证它们与等价关系相容（定义 5.3.4、5.3.9）；
  3. 通过一系列引理证明 构成一个（命题 5.3.11）；
  4. 引入"远离零"概念，构造非零实数的倒数（定义 5.3.12–5.3.16）。

这个构造在数学上完成了从 \(\mathbb{Q}\) 到 \(\mathbb{R}\) 的"完备化"——所有 现在都有了极限（虽然极限本身就是一个）。 在接下来的几节中，我们将在此基础上建立序结构、上确界等进一步的重要性质。

💬 个人感悟：为什么选择 Cauchy 序列？

实数的构造有多种等价的方法：Dedekind 分割、十进制展开、Cauchy 序列。 Tao 选择 Cauchy 序列方法，因为它自然地从前几节的 \(\varepsilon\)-接近概念发展而来， 并且与后续章节中的紧密衔接。

在形式化方面，Cauchy 序列方法也有优势：它直接利用了已的序列概念， 将实数构造归结为在等价关系下取商——这与从 \(\mathbb{N}\) 到 \(\mathbb{Z}\)、 从 \(\mathbb{Z}\) 到 \(\mathbb{Q}\) 的构造在模式上完全一致。 每次都是取某种"形式对象"的等价类，然后在商集上定义运算。

然而，这次有一个根本性区别：前两次构造只需验证代数运算与等价关系相容， 而这一次还需要验证分析性质（Cauchy 性的保持）。这体现了分析学与代数的分野： ——即使只是中的"潜在极限"—— 也带来了全新的复杂性。

📖 补充说明：序列的起始索引问题

Lean 形式化中的一个技术细节：定义 5.1.1 允许序列从任意整数索引开始， 但在第 5.3 节中为了简化商构造，我们要求所有 从 0 开始。这就是 CauchySequence 中 zero : n₀ = 0 字段的作用。

这个约束没有损失一般性：任何从 \(m\) 开始的 Cauchy 序列都可以通过移位 转化为从 0 开始的 Cauchy 序列，且保持等价关系。
